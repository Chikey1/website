Poutine Quest

Purpose

A personal project used to explore interest in mobile development using Android Studio.

Overview  download on Google Play

Poutine Quest is a story based game in which one plays as the protagonist who must overcome three challenges to complete his quest. The three challenges are minigames which the user must complete in order to progress. The app is available on Google Play.

Minigame 1

The place at which the different fries appear is randomly generated. If the player picks the wrong one, the "wrong" screen will flash. If the player fails three times, the "gameover" activity will start. The player can ask for a hint by clicking the lightbulb in the top-right corner.

Minigame 2

If the player picks the wrong combination of equipment, the player will be defeated by the monster and the "gameover" activity will start.

Minigame 3

If the player cannot solve the riddle, the player can choose to either receive a hint or the answer in the top right corner. If the player submits the wrong answer, "wrong answer" will flash above the input field.

Gameover Activity

If at any time the player fails a challenge, they will be brought to the gameover activity. If the player clicks "retry" they will be allowed another attempt at the challenge that they failed. If the player clicks "quit", they will be brought to the "start" screen.

Arduino Keyboard

The keyboard is composed of 5 buttons with frequencies from the C major scale.

Table Tennis Machine

The machine fires ping-ping balls at varying angles.

Ruby QA Script

Purpose

To augment workplace efficiency and productivity by automating website comparisons through source code analysis. Ruby was used to check that the websites contained the same content after migrating to the new cms.

HTML Verifier  view code

The HTML Verifier takes in a domain and uses two different versions of it to access both the old site and new site. All HTML parsing was done using Nokogiri. First, it accessed the sitemap and obtained the relative links of all the pages. Then, by attaching the relative links to the new domain and old domain, it checks that all pages are present. Next, it checks that all <img> on the new site have alt text. Then it compares the class attribute of <img> to the old site and the href attribute of the <a>. Any errors that it detects are outputted to a .txt file. There are also more functionalities such as checking blog posts.

CSS Comparer (Selenium)  view code

The CSS Comparer takes in a domain and uses Selenium to open two windows to access both the old and new site. It uses the sitemap, just like the HTML verifier to traverse the sites. If the sites need a login to view the site, Selenium will send keys to login. Then, it goes page by page to compare CSS values. For example, to compare font styles, it uses a loop to traverse an array containing tags such as <h1>. Then, within that loop, it traverses another array containing attributes such as font-family and compares them to that of the old site. Any inconsistencies will be outputted to a .txt file.

Final Interface  download interface

The above Ruby scripts were then combined with existing Python code which checked other aspects of the sites. Tkinter was used as a graphical interface for greater usability.
